## Re2

### 程序分析

- main函數裡有個RC4加密，解出來是個fake flag...
- 動調後，發現程序來某個地方調用了以下函數來創建新線程
- 在後續的分析中得知這是一道迷宮題，**在調試與不調試的情況下，地圖會有改變，但不知道它是想要調試時的結果還是沒調試時的結果，所以兩種地圖都要試試**

![Untitled](Untitled.png)

- 進入StartAddress，看到它調用了`SetWindowsHookExA`來設置勾子
- 查[文檔](https://learn.microsoft.com/en-us/windows/win32/api/winuser/nf-winuser-setwindowshookexa)，得知`13`代表`WH_KEYBOARD_LL`，即要hook的對象是鍵盤

![Untitled](Untitled1.png)

- 進入`kb_hook_proc`( 鍵盤hook的回調函數 )
- 從這裡就可以看出迷宮題的特徵 ( 注：run_map函數中有個簡單的花指令，patch掉就可以了 )
- 若鍵盤輸入的是W、A、S、D其中一個時，才會進入`run_map`函數去跑迷宮

![Untitled](Untitled2.png)

- 進入`run_map`函數，從這裡可以知道WASD分別對應上下左右，但通過查看`left_right`的交叉引用，發現在`other_run_map`函數中也有類似的操作
- 通過動調，發現是先執行`run_map`再執行`other_run_map`， 傳入`other_run_map`的參數a1對應著`run_map`中`Sleep`函數的參數，例如我輸入了`'A'`，這時會執行`--left_right`和`Sleep(0x13u)`，然後a1=0x13，`++top_bottom`，**即按`'A'`後實際是向左下移動**
- `2`是終點，`1`是不能碰的地方

![Untitled](Untitled3.png)

![Untitled](Untitled4.png)

- 迷宮起點在TLS回調函數中設置，同樣也分為有調試和無調試兩種情況，可以都試試
- 從這裡可以順便學學PEB的反調試，參考文章：[https://www.xuenixiang.com/thread-2729-1-1.html](https://www.xuenixiang.com/thread-2729-1-1.html)

![Untitled](Untitled5.png)

### 腳本

- 40*40的地圖有點大，自己嘗試跑過一遍，結果錯了= =( 看來運氣又沒那麼好了 )

#include<string>

int dirs[4][2] = { {-1,1}, {1,-1}, {1,1}, {-1,-1} }; // a d s w
string PATH = "adsw";

void dfs(char map[],vector<bool> visited,int left_right,int top_bottom,int step,vector<char>& path) {
        if (path.size() > 100 || left_right >39 || top_bottom >39 || map[top_bottom*40+left_right] == 1 )
                return;
        else {

                if (path.size() == 75 && map[top_bottom * 40 + left_right] == 2) {
                        cout << "len = " << path.size() << endl;
                        for (auto& e : path) {
                                cout << e;
                        }
                        cout << endl;
                        return;
                }
        }
        //cout << path.size() << endl;
        for (int i = 0; i < 4;i++) {
                int tmpLR = left_right + dirs[i][0];
                int tmpTB = top_bottom + dirs[i][1];
                if (tmpLR < 0 || tmpLR >= 40 || tmpTB < 0 || tmpTB >= 40)continue;
                if (visited[tmpLR + tmpTB * 40])continue;
                left_right = tmpLR;
                top_bottom = tmpTB;

                visited[left_right + top_bottom * 40] = true;
                step++;
                path.emplace_back(PATH[i]);
                dfs(map, visited, left_right, top_bottom, step,path);
                path.pop_back();
                step--;
                visited[left_right + top_bottom * 40] = false;
                left_right -= dirs[i][0];
                top_bottom -= dirs[i][1];

        }

}
